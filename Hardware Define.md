为了让这 4 台 M5GO 在你们的 L-CASH（无障碍智能家居）项目中各司其职，且不互相干扰，我们需要对它们进行**“解耦”**。一个标准的智能家居系统需要有“感知（传感器）”、“执行（动作器）”和“安防（门禁）”模块。

基于你们手头的 M5GO 硬件生态，我为你们设计了如下的 **4 节点黄金分工方案**：

### 🗺️ L-CASH 硬件 4 节点分工蓝图

#### 📍 Node 1：环境感知中枢（“眼睛与皮肤”）
* **角色定位：** 纯粹的**数据采集者**。它负责不间断地感受房间的环境，并定期向后端“大脑”汇报。
* **推荐外接硬件：** * ENV 传感器（温度、湿度）
    * CO2 / TVOC 传感器（空气质量）
    * 光敏传感器（检测白天/黑夜）
* **功能实现：**
    * 不需要订阅任何指令频道。
    * 每隔 5 秒读取一次传感器数据。
    * 向 `LCASH/sensors` 频道**发布 (Publish)** 数据：`{"temp": 28, "co2": 1200, "light": 45}`。

#### 📍 Node 2：安防与入户门禁（“守门员”）
* **角色定位：** 结合了你之前写过的指纹开门代码，这个节点负责家门的安全与身份识别。
* **推荐外接硬件：**
    * 指纹模块 / RFID 模块（Port C，身份识别）
    * 180度舵机（Port B，模拟门锁）
    * ToF 测距（Port A，防盗防尾随检测）
* **功能实现：**
    * **本地逻辑：** 刷对指纹，直接控制门锁舵机开启。
    * **云端通信：** 每次开门/关门，或者 ToF 测距发现门口有人徘徊，向 `LCASH/status` 发布警报：`{"door_status": "open", "warning": "someone_approaching"}`。
    * **订阅 (Subscribe)：** 监听 `LCASH/commands`，允许大模型或网页端远程一键开锁。

#### 📍 Node 3：全彩氛围照明（“气氛组”）
* **角色定位：** 负责室内的视觉反馈和照明控制，大模型可以根据用户的心情或环境改变灯光。
* **推荐外接硬件：**
    * RGB LED 灯条 / 灯板（Port B 或 Port C）
    * （可选）继电器 Relay（控制一盏真实的台灯）
* **功能实现：**
    * **订阅 (Subscribe)：** 监听 `LCASH/commands`，解析色彩指令。
    * 如果大模型发来：`{"Node3_Light": "WARN", "color": "RED"}`，则全屋亮红灯并闪烁。
    * 如果发来：`{"Node3_Light": "ON", "color": "WARM"}`，则亮起暖色调助眠灯。

#### 📍 Node 4：气候与机械臂（“肌肉”）—— *（我们昨天已完成）*
* **角色定位：** 执行物理机械动作，改善室内环境。
* **外接硬件：**
    * 360度舵机（模拟排风扇）
    * 180/360度舵机（模拟窗帘拉伸）
* **功能实现：** 监听 `LCASH/commands`，根据大模型指令开关风扇和窗帘，执行完毕后发布状态变更。

### ⚙️ 具体怎么实现？（核心通信协议）

要想让 4 台机子完美配合你的 Python + 大模型后端，**最关键的是管好 MQTT 的通信频道和 JSON 格式**。千万不要让所有机子乱发消息。

你需要建立一张这样的“通信契约表”，发给前端、后端和硬件同学：

| 设备节点 | MQTT 动作 | 目标主题 (Topic) | 传输的 JSON 格式示例 |
| :--- | :--- | :--- | :--- |
| **Node 1 (感知)** | 持续发布 (上报) | `LCASH/sensors` | `{"node": 1, "temp": 28, "hum": 60, "co2": 1500}` |
| **Node 2 (门禁)** | 发布 (状态/报警) | `LCASH/status` | `{"node": 2, "door": "unlocked", "warning": null}` |
| **Node 2 (门禁)** | 订阅 (听指令) | `LCASH/commands` | `{"Node2_Door": "OPEN"}` |
| **Node 3 (灯光)** | 订阅 (听指令) | `LCASH/commands` | `{"Node3_Light": "ON", "color": "BLUE"}` |
| **Node 4 (气候)** | 订阅 (听指令) | `LCASH/commands` | `{"Node4_Servo_Fan": "ON", "Node4_Curtain": "OPEN"}` |
| **Node 4 (气候)** | 发布 (状态反馈) | `LCASH/status` | `{"node": 4, "fan": true, "curtain": true}` |

### 🧠 后端大模型如何调配它们？

昨天的 Python 代码里，我们只给了大模型 Node 4 的权限。现在你只需要在 `app.py` 的 **System Prompt（系统提示词）**里把 4 个节点都告诉它：

> **给大模型的提示词补充：**
> "你现在拥有整个房子的控制权。
> - 门禁系统 (Node2_Door): 可选 'OPEN' / 'CLOSE'
> - 灯光系统 (Node3_Light): 可选 'ON' / 'OFF', 附加颜色
> - 排风系统 (Node4_Servo_Fan): 可选 'ON' / 'OFF'
>
> 如果用户说：'我回家了，屋里好黑，还有点闷。' 
> 你应该返回：`{"executed_actions": {"Node2_Door": "OPEN", "Node3_Light": "ON", "Node4_Servo_Fan": "ON"}}`"

这样，Python 收到指令后，原封不动地发到 `LCASH/commands`。
* Node 2 看到有 `Node2_Door`，立刻开门。
* Node 3 看到有 `Node3_Light`，立刻开灯。
* Node 4 看到有 `Node4_Servo_Fan`，立刻转风扇。
整个系统就像一个交响乐团一样完美运转！
