这是一个非常关键的环节。在 IoT 项目中，**LLM（大模型）充当的是“大脑”的角色**，而传感器和执行器则是它的“五官”和“四肢”。

为了让负责 LLM 设计的同学能写出精准的逻辑，你需要向他提供一份详尽的**“硬件能力清单”**和**“数据契约”**。

### 第一部分：L-CASH 项目硬件功能汇总（当前进度）

目前你的项目已经通过 4 个 Node 节点实现了**“感知-决策-反馈”**的全链路：

1.  **环境感知模块 (Node 1)**
    * **传感器**：CO2/TVOC 传感器、光照传感器。
    * **功能**：实时监测室内空气质量和光强度，数据每秒上传至 MQTT `LCASH/sensors` 频道。
    * **意义**：为 LLM 提供“家里闷不闷”和“光线够不够”的判断依据。

2.  **微气候执行模块 (Node 2)**
    * **传感器/执行器**：ENV 温湿度传感器、舵机（窗帘）、物理按钮。
    * **功能**：上报温湿度；接收并执行 `FAN:ON/OFF` 和 `CURTAIN:ON/OFF` 指令。
    * **意义**：LLM 的“手脚”，负责物理调节环境。

3.  **安防门禁模块 (Node 3)**
    * **传感器**：ToF 测距传感器、指纹识别模块。
    * **功能**：检测门口是否有异常靠近（`<300mm` 触发报警）；识别合法主人身份并解锁。
    * **意义**：提供“安全上下文”，让 LLM 知道此时家里的状态（警戒中、主人已回、有非法靠近）。

4.  **交互反馈模块 (Node 4)**
    * **传感器/执行器**：手势传感器、NeoPixel RGB 灯带。
    * **功能**：通过手势切换灯光氛围；通过灯带颜色提供视觉反馈。
    * **意义**：提供非语言的交互方式，LLM 也可以通过控制灯带颜色来表达系统状态（如：指令成功变绿，失败变红）。

### 第二部分：给 LLM 同学的“设计指南”

为了让 LLM 能够智能地调度上述硬件，你需要给 LLM 同学汇总以下 4 个核心要点：

#### 1. 明确的“动作指令集”（Action Space）
LLM **只能**输出以下指令，严禁生成其他格式，否则硬件无法识别：
* `FAN:ON` / `FAN:OFF` —— 控制风扇（排风/降温）
* `CURTAIN:ON` / `CURTAIN:OFF` —— 控制窗帘（遮阳/通风）
* *(预留)* `LED:COLOR_HEX` —— 如果未来需要 LLM 控制氛围灯。

#### 2. 环境参数的“阈值参考”
LLM 需要根据数据判断何时该动作，可以给他设定一些标准参考值：
* **温度**：高于 28°C 建议开启 `FAN:ON`。
* **CO2**：高于 1000ppm 建议开启 `FAN:ON`（排风）。
* **光照**：若用户说“太刺眼了”或光照 > 800lux，建议 `CURTAIN:OFF`（关窗帘）。

#### 3. 情境感知的逻辑逻辑 (Context Awareness)
告诉 LLM 应该结合多个传感器来做复杂判断，而不仅仅是关键词匹配：
* **案例 A**：用户说“我回来了”。
    * *硬件输入*：Node 3 发送了 `unlock_success`。
    * *LLM 逻辑*：回复“欢迎回家”，并自动执行 `CURTAIN:ON` (开帘迎客) + `FAN:ON` (换气)。
* **案例 B**：用户说“我要睡觉了”。
    * *LLM 逻辑*：执行 `CURTAIN:OFF` + `FAN:OFF`，并回复“晚安，已为您营造睡眠环境”。

#### 4. 接口协议格式（API Contract）
LLM 输出的结果必须严格封装在以下 JSON 结构中：
```json
{
  "ai_reply": "这是显示在网页上的温柔话语",
  "executed_actions": ["指令1", "指令2"],
  "thought": "LLM 内部的思考过程，不在前端显示，方便调试"
}
```

### 第三部分：建议 LLM 同学实现的“高级功能”

你可以给设计 LLM 的同学提几个更具挑战性的方向：
* **模糊指令处理**：当用户说“家里有点闷”，LLM 应能通过 CO2 数据判断出是该开窗还是开扇。
* **异常拦截**：如果 Node 3 处于 `intrusion_alert` (报警状态)，LLM 应该优先处理安防回复，拒绝执行普通的开关指令。
* **主动建议**：如果传感器检测到 CO2 爆表，即使主人没说话，LLM 也可以在网页端主动询问：“监测到空气质量不佳，是否为您开启排风扇？”

### 🤖 给 LLM 同学的开发指南：如何指挥我们的家

你好！我是负责硬件端和 Web 端设计的。目前我们的“身体”（硬件传感器和执行器）已经全部就绪，现在需要你为它设计一个“大脑”。

#### 1. 核心交互契约（JSON 格式）
当用户在网页上输入一句话时，你的 LLM 需要输出如下格式的 JSON。**请严格遵守这个格式，因为后端 Flask (app.py) 会根据这个结构自动分发指令：**

```json
{
  "ai_reply": "这是你用温柔、人性化的语气回复用户的话",
  "executed_actions": ["ACTION_CODE_1", "ACTION_CODE_2"],
  "thought": "（可选）简述你为什么要执行这些动作，方便我们调试"
}
```

#### 2. 你可以下达的“死指令”（Action Space）
硬件目前只能听懂以下四个特定的字符串，**请不要改写它们**：

| 动作代码 | 物理效果 | 适用场景建议 |
| :--- | :--- | :--- |
| **`FAN:ON`** | 开启空气循环扇 | 室内闷热、CO2过高、用户说“想吹风” |
| **`FAN:OFF`** | 关闭空气循环扇 | 室内凉爽、用户离开、用户说“关掉风扇” |
| **`CURTAIN:ON`** | 物理开启窗帘 | 早晨起床、阳光好、用户说“透透气” |
| **`CURTAIN:OFF`** | 物理关闭窗帘 | 晚上睡觉、看电影、阳光刺眼、用户说“遮光” |

---

### 3. 你能拿到的“感知数据”（Input Context）
在调用你的函数时，我会把当前的传感器数据传给你，你可以根据这些数据做“自主决策”：

* **`temp` (温度)**：单位 °C。超过 28°C 建议主动建议开风扇。
* **`co2` (二氧化碳)**：单位 ppm。超过 1000ppm 说明空气浑浊，必须开风扇。
* **`light` (光照)**：单位 lux。数值大代表阳光强烈。
* **`event` (安防事件)**：
    * `unlock_success`：主人指纹解锁成功了。
    * `intrusion_alert`：检测到门口有陌生人长时间逗留。

### 4. 给 LLM 的场景训练词（Prompting Hints）

为了让你的 LLM 表现得更像一个聪明、温馨的管家，请训练它识别以下场景词，并作出对应的硬件联动：

#### **场景 A：温馨回家 (Identity Recognition)**
* **用户输入**：无（由系统自动触发 `unlock_success`）
* **LLM 逻辑**：
    * `ai_reply`: "欢迎回家，主人！已经为您打开窗帘和通风扇，享受舒适的午后吧。"
    * `executed_actions`: `["CURTAIN:ON", "FAN:ON"]`



#### **场景 B：空气管理 (Air Quality Control)**
* **数据背景**：`co2: 1250`
* **用户输入**：随便什么，或者用户问“家里环境怎么样？”
* **LLM 逻辑**：
    * `ai_reply`: "目前室内二氧化碳浓度略高，有点闷，我为您开启了排风系统。"
    * `executed_actions`: `["FAN:ON"]`

#### **场景 C：沉浸式电影模式 (Ambient Mode)**
* **用户输入**： "我想看场电影" 或 "太刺眼了"
* **LLM 逻辑**：
    * `ai_reply`: "好的，已为您关闭窗帘，营造私人影院氛围。"
    * `executed_actions`: `["CURTAIN:OFF"]`

#### **场景 D：模糊指令处理 (Vague Request)**
* **用户输入**： "我感觉不太舒服"
* **LLM 逻辑**：
    * (检查数据) 若温度高 -> 建议开扇；若二氧化碳高 -> 建议通风。
    * `ai_reply`: "检测到室内温度偏高，已为您开启降温。需要我为您联系家人吗？"

### 5. 给 LLM 同学的小建议
1.  **语气要温柔**：我们现在的 UI 是现代简约温馨风格，所以 AI 的回复（`ai_reply`）应该像个老朋友，而不是机器人。
2.  **多指令并行**：`executed_actions` 是个列表，你可以一次性下达 `["FAN:OFF", "CURTAIN:OFF"]`。
3.  **安全优先**：如果 `event` 收到 `intrusion_alert`，AI 应该优先提醒用户查看监控，而不是去管风扇。
